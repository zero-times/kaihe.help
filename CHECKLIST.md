## 四、统一内容规范

### 页面模板
- [ ] 每页标题直接对应搜索问题
- [ ] 每页开头先给结论
- [ ] 每页主体按步骤展开
- [ ] 每页包含“哪些先别做/常见误区”
- [ ] 每页包含相关推荐
- [ ] 每页标注更新时间

### 文风与表达
- [ ] 不写成品牌宣传稿
- [ ] 不写成长篇法律论文
- [ ] 不使用过度情绪化表达
- [ ] 尽量使用短句和可执行语句
- [ ] 优先写“现在先做什么”
- [ ] 保持标题、术语、表达统一

## 六、上线前检查

### 可用性检查
- [ ] 手机端首页首屏清晰
- [ ] 所有页面都能正常打开
- [ ] 所有内部链接都无报错
- [ ] HTTPS 正常
- [ ] 页面加载速度可接受（Lighthouse Performance > 90）
- [ ] 404 页面可回首页
- [ ] GitHub Actions 构建部署正常

### 无障碍检查
- [ ] 正文字号 ≥ 16px
- [ ] 文字与背景色对比度 ≥ 4.5:1
- [ ] 可点击元素有明确视觉反馈
- [ ] 页面使用语义化 HTML（main / nav / article / section）
- [ ] 图片提供 alt 文本

### 性能检查
- [ ] 首页首次加载体积 < 200KB
- [ ] 页面无前端 JavaScript 运行时
- [ ] 未使用大体积 Web 字体（使用系统字体栈）
- [ ] 图片已压缩（SVG / WebP 优先）

### 内容检查
- [ ] 首页信息足够直接
- [ ] 核心页面都能独立成立
- [ ] 没有明显空白页
- [ ] 没有占位文案未替换
- [ ] 没有明显错别字或断链

### 风险边界检查
- [ ] 不出现收费/商业服务文案
- [ ] 不出现代举报/代维权承诺
- [ ] 不出现夸大法律效果的表述
- [ ] 不出现可能引发误解的”官方”措辞
- [ ] 保持个人知识站/指南站定位
- [ ] 不出现”保证删帖””百分百有效”等禁止表达

### 上线前内容审核
- [ ] 对照页面模板规范逐页检查
- [ ] 对照禁止表达列表逐页检查
- [ ] 以”刚被开盒的慌张用户”视角通读每个核心页面
- [ ] 确认每个页面 frontmatter 完整（title / description / pubDate / relatedPages）
